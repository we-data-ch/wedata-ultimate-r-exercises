Introduction to R

R is a powerful programming language for statistical computing and data analysis. Originally developed by statisticians Ross Ihaka and Robert Gentleman at the University of Auckland in the 1990s, R has become a leading tool for data science, statistical analysis, and data visualization.

Key Features of R:

  • Statistical Focus: R was built specifically for statistical analysis and offers a vast array of statistical methods and tests.
  • Graphics Capabilities: Excellent for creating high-quality visualizations with packages like ggplot2.
  • Extensibility: Thousands of packages available through CRAN (Comprehensive R Archive Network) extend R’s functionality.
  • Open Source: Free and open-source, with a strong community of users and developers.
  • Interactive Environment: The R console provides immediate feedback, making it ideal for data exploration.

The R Console and Basic Operations

When you start R, you’re presented with a console where you can enter commands. The console interprets each command and returns results immediately.

R functions as a sophisticated calculator, capable of handling basic arithmetic operations and much more. Here are some basic operations you can perform:

# Addition
5 + 3  # Returns 8

# Subtraction
10 - 4  # Returns 6

# Multiplication
6 * 7  # Returns 42

# Division
20 / 5  # Returns 4

# Exponentiation (Power)
2^3  # Returns 8

Comments and Output in R

R allows you to add comments to your code to explain what it does. Comments start with the # symbol and are ignored by R when executing code.

R automatically prints the result of a calculation or command if it’s not assigned to a variable. This feature is useful for quick calculations and checking intermediate results.

You can also explicitly print values using the print() function, which is especially useful when working with scripts where automatic printing doesn’t occur.

# This is a comment that R ignores

# Automatic printing example
42  # R will output 42

# Print function example
print("Hello")  # Explicitly prints "Hello"

Names and Assignment

In R, you can store values in variables for later use. This is done using the assignment operator, which can be either <- or =. The <- operator is more commonly used in R programming.

Variable Assignment

When you assign a value to a variable, R stores that value in memory. You can then use the variable name to refer to that value in subsequent operations.

# Assign value 10 to variable x
x <- 10

# Use x in a calculation
x + 5  # Returns 15

# Reassign x to a new value
x <- 20
x  # Now returns 20

While both <- and = work for assignment in most contexts, <- is preferred in R programming because it clearly distinguishes assignment from the == equality test and because it works in all contexts.

Variable Names

Good variable names are essential for readable code. In R:

  • Variable names can contain letters, numbers, dots (.), and underscores (_)
  • Names must start with a letter or a dot not followed by a number
  • Names are case-sensitive (count and Count are different variables)
  • Names should be descriptive of what they contain
# Valid variable names
age <- 30
student.name <- "John"
test_score <- 95

# Invalid variable names (would cause errors)
# 2ndPlace <- "Silver"  # Can't start with a number
# test-score <- 95      # Can't contain hyphens

Data Types in R

R has several basic data types:

  1. Numeric: Decimal numbers (1.5, 42.0)
  2. Integer: Whole numbers (1L, 42L, where the L indicates an integer)
  3. Character: Text, enclosed in quotes ("hello", 'R is fun')
  4. Logical: Boolean values (TRUE or FALSE)
  5. Complex: Complex numbers with real and imaginary parts (1+2i)
  6. Date: Date values (as.Date("2023-01-15"))

You can check the type of any variable using the class() function.

# Numeric
x <- 10.5
class(x)  # Returns "numeric"

# Integer
y <- 10L
class(y)  # Returns "integer"

# Character
name <- "R Language"
class(name)  # Returns "character"

# Logical
is_valid <- TRUE
class(is_valid)  # Returns "logical"

# Date
today <- as.Date("2023-01-15")
class(today)  # Returns "Date"

Vectors

Vectors are one of the most basic and important data structures in R. A vector is a sequence of data elements of the same type. It’s like a collection of values stored together.

Creating Vectors

There are several ways to create vectors in R:

  1. Using the c() function (combine function):

    # Create a numeric vector
numbers <- c(1, 2, 3, 4, 5)

# Create a character vector
colors <- c("red", "green", "blue")

  2. Using the colon operator : for sequences:

    # Create a sequence from 1 to 10
seq_1_10 <- 1:10  # Contains 1, 2, 3, ..., 10

  3. Using the seq() function for more complex sequences:

    # Create a sequence from 0 to 1 with 0.1 steps
seq_0_1 <- seq(0, 1, by = 0.1)  # 0, 0.1, 0.2, ..., 1

  4. Using the rep() function to repeat values:

    # Repeat the value 5 four times
rep_values <- rep(5, times = 4)  # 5, 5, 5, 5

Vector Types

All elements in a vector must be of the same type. The main vector types are:

  • Numeric vectors: contain numbers (c(1.5, 2.0, 3.5))
  • Integer vectors: contain whole numbers (c(1L, 2L, 3L))
  • Character vectors: contain text (c("apple", "banana"))
  • Logical vectors: contain TRUE or FALSE values (c(TRUE, FALSE, TRUE))
  • Date vectors: contain date values

Vector Operations

R is designed to work efficiently with vectors. Most operations are “vectorized,” meaning they apply to each element of the vector automatically.

Arithmetic with Vectors

When you perform arithmetic operations on vectors, R applies the operation to each element:

# Vector addition
c(1, 2, 3) + c(4, 5, 6)  # Returns c(5, 7, 9)

# Vector multiplication
c(1, 2, 3) * 2  # Returns c(2, 4, 6)

Vector Recycling

If vectors have different lengths, R “recycles” the shorter vector to match the length of the longer one:

# Vector of length 3 plus vector of length 1
c(1, 2, 3) + 10  # Returns c(11, 12, 13)

# Vector of length 3 plus vector of length 2
c(1, 2, 3) + c(10, 20)  # Returns c(11, 22, 13), with warning

Vector Functions

R has many built-in functions for working with vectors:

x <- c(2, 4, 6, 8, 10)

length(x)    # Returns 5 (number of elements)
sum(x)       # Returns 30 (sum of all elements)
mean(x)      # Returns 6 (average of elements)
median(x)    # Returns 6 (median value)
min(x)       # Returns 2 (minimum value)
max(x)       # Returns 10 (maximum value)
sort(x)      # Returns sorted vector
rev(x)       # Returns reversed vector

Operators

Operators are symbols that tell R to perform specific mathematical or logical operations. R has a rich set of operators that allow you to perform various calculations and comparisons.

Arithmetic Operators

Arithmetic operators perform mathematical calculations:

Operator Description Example Result
+ Addition 5 + 3 8
- Subtraction 5 - 3 2
* Multiplication 5 * 3 15
/ Division 5 / 3 1.666667
^ Exponentiation 5 ^ 3 125
%% Modulus (remainder) 5 %% 3 2
%/% Integer Division 5 %/% 3 1

These operators can work on single values or vectors, applying the operation to each element of the vector.

# Vector arithmetic
c(1, 2, 3) + c(4, 5, 6)  # Returns c(5, 7, 9)
c(1, 2, 3) * 2           # Returns c(2, 4, 6)

Comparison Operators

Comparison operators compare values and return logical (TRUE/FALSE) results:

Operator Description Example Result
== Equal to 5 == 3 FALSE
!= Not equal to 5 != 3 TRUE
< Less than 5 < 3 FALSE
> Greater than 5 > 3 TRUE
<= Less than or equal to 5 <= 5 TRUE
>= Greater than or equal to 5 >= 6 FALSE

These operators are essential for filtering data and controlling program flow with conditionals.

Logical Operators

Logical operators combine logical (TRUE/FALSE) values:

Operator Description Example Result
& Element-wise AND c(TRUE, FALSE) & c(TRUE, TRUE) c(TRUE, FALSE)
| Element-wise OR c(TRUE, FALSE) | c(FALSE, TRUE) c(TRUE, TRUE)
! NOT !TRUE FALSE
&& AND (single value) TRUE && FALSE FALSE
|| OR (single value) TRUE || FALSE TRUE

The element-wise operators (&, |) work on vectors, while the single-value operators (&&, ||) only evaluate the first element of vectors.

# Element-wise AND
c(TRUE, FALSE, TRUE) & c(TRUE, TRUE, FALSE)  # Returns c(TRUE, FALSE, FALSE)

# Single-value AND (uses only first element)
c(TRUE, FALSE, TRUE) && c(TRUE, TRUE, FALSE)  # Returns TRUE

Factors and Lists

Factors

Factors are a special type of vector used to represent categorical data. They are particularly useful in statistical modeling and data visualization.

Creating Factors

Factors are created using the factor() function. By default, R will sort the levels alphabetically, but you can specify a custom order if needed.

# Create a basic factor
gender <- factor(c("Male", "Female", "Female", "Male", "Male"))
gender  # Levels: Female Male

# Specify custom levels (including order)
blood_type <- factor(c("A", "B", "AB", "O", "A"), 
                     levels = c("O", "A", "B", "AB"))
blood_type  # Levels: O A B AB

Working with Factors

Factors are useful for:

  1. Ensuring data consistency: Only values in the defined levels are allowed
  2. Controlling the order of categories: Important for plots and tables
  3. Efficient storage: Factors store integers internally with a lookup table
  4. Statistical modeling: Proper handling of categorical variables

Common functions used with factors:

# Get the levels of a factor
levels(gender)  # Returns "Female" "Male"

# Count occurrences of each level
table(gender)  # Shows frequency table

# Convert factor to numeric
as.numeric(blood_type)  # Returns the underlying integer codes

# Convert factor to character
as.character(blood_type)  # Returns the original strings

Lists

Lists are a versatile data structure in R that can hold elements of different types, including other lists. This makes them ideal for hierarchical or heterogeneous data.

Creating Lists

Lists are created using the list() function. Each element can have a name for easier access.

# Create a basic list
my_list <- list(100, "Hello", TRUE, c(1, 2, 3))

# Create a named list
student <- list(
  name = "John Smith",
  age = 21,
  grades = c(85, 92, 78, 90),
  passed = TRUE
)

Accessing List Elements

There are multiple ways to access elements in a list:

  1. Using double square brackets [[]] for a single element:

    student[[1]]        # Returns "John Smith"
student[["name"]]   # Returns "John Smith"

  2. Using the dollar sign $ for named elements:

    student$name        # Returns "John Smith"
student$grades[2]   # Returns 92 (second element of grades)

  3. Using single square brackets [] to get a sublist:

    student[1:2]        # Returns a list with name and age
student["grades"]   # Returns a list with just the grades element

Modifying Lists

Lists are mutable, so you can change them after creation:

# Add a new element
student$email <- "john@example.com"

# Change an existing element
student$age <- 22

# Remove an element
student$passed <- NULL

Data Frames

Data frames are the most common data structure for tabular data in R. They’re similar to tables in a database, spreadsheets, or CSV files.

Key Features of Data Frames

  • Rectangular data (rows and columns)
  • Each column can be a different data type
  • All columns must have the same length
  • Each column has a name
  • Rows can be named (but often aren’t)

Creating Data Frames

Data frames are created using the data.frame() function:

# Create a data frame
students <- data.frame(
  name = c("Alice", "Bob", "Charlie", "David"),
  age = c(22, 25, 21, 24),
  grade = c(85, 92, 78, 95),
  pass = c(TRUE, TRUE, FALSE, TRUE)
)

You can also create data frames from external sources using functions like: - read.csv() - read from CSV files - read.table() - read from text files - read_excel() - read from Excel files (requires the readxl package)

Examining Data Frames

R provides several functions to examine data frames:

# View the structure of a data frame
str(students)

# Show the first few rows
head(students)

# Show the last few rows
tail(students)

# Get a summary of the data frame
summary(students)

# Get the dimensions (rows, columns)
dim(students)

# Get the column names
names(students)

Accessing Data Frame Elements

You can access elements of a data frame in several ways:

1. Using column names with the $ operator:

# Get an entire column
students$name  # Returns c("Alice", "Bob", "Charlie", "David")

# Get a specific element
students$age[2]  # Returns 25 (Bob's age)

2. Using square brackets [row, column]:

# Get a specific cell (row 2, column 3)
students[2, 3]  # Returns 92 (Bob's grade)

# Get multiple rows/columns
students[1:2, c("name", "age")]  # First two students, name and age only

# Get all rows of a column
students[, "name"]  # All names

# Get all columns for a row
students[2, ]  # All data for Bob

3. Filtering data frames with logical conditions:

# Get rows where age is over 22
students[students$age > 22, ]

# Get rows where student passed
students[students$pass == TRUE, ]

# Combine conditions
students[students$age > 22 & students$pass == TRUE, ]

Modifying Data Frames

Data frames can be modified after creation:

# Add a new column
students$email <- c("alice@example.com", "bob@example.com", 
                   "charlie@example.com", "david@example.com")

# Change a value
students$age[3] <- 22

# Add a new row
new_student <- data.frame(
  name = "Eve",
  age = 23,
  grade = 88,
  pass = TRUE,
  email = "eve@example.com"
)
students <- rbind(students, new_student)
